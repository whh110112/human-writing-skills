# Editor Loop Skill

## Aim

Use an iterative workflow instead of expecting one perfect generation. Draft, diagnose, rewrite locally, then run a final continuity and rhythm pass.

## Workflow

1. Draft
   - Produce the requested passage with the selected style and context ledger.

2. Diagnose
   - Mark lines that feel too smooth, generic, explanatory, emotionally flat, or discontinuous.
   - Name the issue; do not merely say "sounds AI".

3. Rewrite Locally
   - Rewrite only the weak lines or paragraphs.
   - Preserve facts, timeline, and user constraints.

4. Re-score
   - Evaluate the revised passage with the AI Trace Rubric.
   - If two or more dimensions remain weak, run one more local rewrite.

5. Final Output
   - Return the polished passage.
   - Include notes only when the user asks for them.

## Avoid

- rewriting the whole piece when only three lines are weak
- adding quirks that break genre or narrator
- making the prose messy just to seem human
- changing established facts during revision
