# Reference Style Alignment

## Activation Gate

Use this module only when the user supplies reference files, an explicit style
description, or a direct instruction such as "参考某种文风", "贴近这段写法",
"match this sample's voice", or "write in the style described below".

If none of those signals exists, do not infer a reference style and do not activate
this module.

## Aim

Transfer high-level writing characteristics from supplied material without importing
its plot facts, unique imagery, private information, or distinctive phrasing.

## Style Fingerprint

Extract evidence for these dimensions before drafting or auditing:

- Narrative distance and point of view
- Sentence-length range, pauses, fragments, and paragraph weight
- Lexical register, formality, repetition tolerance, dialect, and taboo phrasing
- Image system: concrete/abstract balance, sensory channels, metaphor frequency
- Scene description order: space -> object -> action, or another recurring sequence
- Character description: action, body, clothing, interior thought, social observation
- Dialogue cadence: turn length, interruption, silence, address forms, subtext
- Emotional method: label, embodiment, contradiction, restraint, or retrospective comment
- Transition method and closure pressure
- Genre permissions and deliberate imperfections

For every claimed trait, cite a short feature or location from the reference. Do not
invent a trait because it would sound attractive.

## Separation Rules

- Treat project context and continuity ledgers as factual authority.
- Treat reference material as style evidence only unless the same file is explicitly
  supplied as project context.
- Do not copy names, events, world rules, relationships, arguments, or conclusions.
- Do not reuse distinctive phrases, signature metaphors, or recognizable sentence chains.
- When only a named style is supplied and no sample exists, convert it into broad,
  non-exclusive traits. Do not claim access to unseen text.
- For a living author or a highly distinctive requested voice, use a high-level trait
  description rather than close imitation.

## Drafting Workflow

1. Build a compact style card with 5 to 10 evidence-backed traits.
2. Mark which traits fit the selected genre and current task.
3. Draft from project facts while applying those traits.
4. Compare the result against the style card, not against memorable phrases.
5. Remove accidental borrowing and restore the current characters' established voices.

## Audit Workflow

Classify each dimension as:

- aligned
- partly aligned
- unsupported by the reference
- conflicting with project context or character voice

Quote or locate evidence from both the reference and the draft. Recommend local repairs
instead of replacing the whole passage unless drift is global.

## Output

When notes are requested:

```text
Reference Style Card
- Trait:
- Reference evidence:
- Transfer rule:
- Do-not-copy boundary:

Style Match Audit
- Aligned dimensions:
- Drift with draft evidence:
- Context or character conflicts:
- Minimal repair:
```

When the user asks only for prose, apply the style card silently.
