# 数字必要性审查

AI 写作常见一种“假精确感”：在情绪、动作、身体反应、亲密距离、停顿里写出过于精确的数字。

例如：

- 手指往上移动了 3 厘米
- 他沉默了 7 秒
- 她后退了 1 厘米
- 两人相距 2.3 米

这些数字看似具体，但如果没有人测量、没有证据功能、没有人物职业习惯，读起来会像模型在拿尺子写小说。

## 但不要删除所有数字

有些数字必须保留：

- 建筑高度：楼高 476 米
- 医学/法医鉴定：伤口 2.3 厘米
- 工程尺寸、技术参数、实验数据
- 年龄、日期、房号、车次、价格、剂量、比分
- 证据、线索、倒计时、规则、成本、边界条件

## 推荐用法

生成时：

```powershell
python -m humanwriting.cli build `
  --style fiction `
  --review `
  --number-sense `
  --task "写一段人物情绪紧张的对话，避免不必要的精确数字。"
```

小说、网文、自媒体在 `--review` 时会默认加入 `natural-measurement`。新闻、科研、技术报告不会默认加入，以免把必要数字弱化。

审稿时：

```powershell
python -m humanwriting.cli audit `
  --draft examples/false-precision-draft.zh-CN.md `
  --numbers `
  --no-strict-continuity
```

## 审查标准

每个精确数字都要问：

- 谁在当下知道这个数字？
- 是测量、记录、新闻、报告、鉴定、工程参数，还是叙述者凭空给的？
- 数字是否改变证据、风险、剧情、人物判断或世界设定？
- 如果换成感官尺度、关系尺度、模糊尺度，是否更自然？

## 处理方式

| 类型 | 处理 |
| --- | --- |
| 公共事实/技术事实 | 保留 |
| 医学、法医、工程、新闻、科研数字 | 保留 |
| 人物职业或性格导致的精确观察 | 可保留 |
| 情绪动作里的微小精确距离 | 弱化 |
| 无意义的精确停顿和次数 | 改成节奏或效果 |

## 示例

| 原句 | 修改 |
| --- | --- |
| 她的手指往上移动了三厘米。 | 她的手指沿着杯沿往上挪了一点。 |
| 他沉默了七秒。 | 他沉默得太久，久到她先移开了眼。 |
| 她后退了 1 厘米。 | 她退了一点，刚好退到灯照不到的地方。 |
| 伤口长 2.3 厘米。 | 保留。医学/鉴定语境需要精确。 |
