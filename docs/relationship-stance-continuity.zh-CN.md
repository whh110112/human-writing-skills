# 关系立场连续性

空间连续性解决“人在不在这个位置”。关系立场连续性解决“这个人会不会在这个场合这样提另一个人”。

很多长文混乱并不是人物关系没写，而是 AI 没有维护：

- 谁和谁敌对
- 谁和谁表面友好、私下不和
- 谁知道秘密，谁只能怀疑
- 谁不能知道另一个人的存在
- 谁在谁面前不能夸、不能骂、不能比较、不能随口提

## 核心方法：对话三角

审每句对话时，不只看说话双方，而是抽取：

```text
说话人 -> 听话人/在场观众 -> 被提及的第三人/组织/秘密
```

然后检查：

- 说话人是否知道听话人与第三方的关系？
- 这个听话人面前是否适合提第三方？
- 语气是否符合公开关系和私下目的？
- 有没有泄露不该泄露的信息？
- 如果违反常规，文本是否给出动机或后果？

## 通用关系账本字段

| 字段 | 用途 |
| --- | --- |
| public stance | 公开姿态：尊敬、敌对、中立、亲密、上下级 |
| private stance | 私下立场：忠诚、怨恨、利用、害怕、隐瞒 |
| knowledge | 知道、怀疑、误解、绝不能知道 |
| mention policy | 能不能提、怎么提、能不能夸/骂/比较 |
| exposure risk | 一旦说漏，风险低/中/高 |
| leverage | 说出这件事会带来什么筹码、损失或威胁 |

## 三类常见错误

### 1. 敌对/不对付关系

董事长 A 和总经理 B 不对付。主角在 A 面前替 B 说好话，或者在 B 面前随口夸 A，必须有明确动机：

- 打小报告
- 试探反应
- 挑拨
- 求情
- 撇清关系
- 装作中立

没有动机就应判为关系立场错误。

### 2. 出轨/多角关系

女 A 与男 A、男 B 都有隐藏关系。她不应无故让二者知道彼此存在，也不应在其中一人面前自然谈另一个人的细节。

除非文本明确写出：

- 她故意刺激对方
- 她说漏嘴
- 对方已经发现
- 她用假身份或代称遮掩
- 这段关系已经公开

否则应判为信息泄露或对象比较错误。

### 3. 门派、家族、公司、阵营关系

两个掌门关系密切时，门人在公开场合无故诋毁对方，应有后果。两个掌门敌对时，门人在本派掌门面前替对方说好话，也应有策略或风险。

同理适用于：

- 公司高层派系
- 后宫/家族关系
- 帮派、宗门、朝堂
- 战队、办公室、学校小团体

## 推荐命令

`--deep-review` 会包含 `relationship-stance-audit`；也可以单独用 `--module relationship-stance-audit`：

```powershell
python -m humanwriting.cli build `
  --style webnovel `
  --deep-review `
  --context examples/relationship-stance-ledger.zh-CN.md `
  --task "续写宴会对话。任何人提到敌对阵营或隐藏关系时，都必须符合在场听众和信息权限。"
```

审已有稿件：

```powershell
python -m humanwriting.cli audit `
  --draft examples/problem-car-scene-draft.zh-CN.md `
  --context examples/relationship-stance-ledger.zh-CN.md `
  --profile relationship
```

## 最小修复原则

发现关系立场错误时，不要整段重写。优先选择：

- 换听众
- 换被比较对象
- 把真名改成代称或暗语
- 增加一句动机
- 增加一个后果
- 更新账本，说明关系已经公开或发生变化
