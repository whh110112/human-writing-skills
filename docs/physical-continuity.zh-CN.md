# 物理连续性：防止人物瞬移和日常细节漂移

AI 长文写作常见的问题不是剧情忘了，而是物理状态忘了：

- 前排人物忽然出现在后排
- 后排人物突然能轻松碰到前排物品
- 人物刚才穿短裙，后面变成长裙
- 平底鞋无过渡变成高跟鞋
- 手机、包、信封、伤口突然消失

解决办法是把“空间”和“物品”从普通剧情摘要里拆出来，单独维护物理状态账本。

## 推荐使用

```powershell
python -m humanwriting.cli build `
  --style fiction `
  --strict-continuity `
  --review `
  --context examples/vehicle-scene-ledger.md `
  --task "续写车内争执。任何座位变化都必须写出动作过渡，保持服装和道具状态一致。"
```

`--strict-continuity` 会自动加入：

- `spatial-blocking`
- `appearance-prop-continuity`
- `physical-continuity-audit`

## 车辆场景必须记录什么

| 项目 | 示例 |
| --- | --- |
| 座位 | driver seat, front passenger seat, rear-left, rear-right |
| 朝向 | 面向前方、转向窗外、通过后视镜看见 |
| 可触达范围 | 方向盘、手套箱、车门、身边的人 |
| 障碍 | 安全带、中央扶手、前排座椅、狭窄过道 |
| 移动过渡 | 解开安全带、停车、下车绕到另一侧、挤过膝盖 |

## 写作规则

- 没有移动过渡，就保持上一位置。
- 没有换装过渡，就保持上一服装。
- 没有放下/交出/丢失动作，道具仍在原处。
- 车在行驶时，座位变化必须极其困难，不能轻描淡写。
- 人物能不能看见、碰到、挡住别人，要由座位和朝向决定。

## 审稿口令

可以在 Chatbox 或其他模型里输入：

```text
只做物理连续性审查，不要续写。

检查：
- 每个人当前在哪个位置
- 谁在前，谁在后，谁在左，谁在右
- 是否有人无过渡换座位
- 服装、鞋子、发型、伤口是否漂移
- 道具是否凭空出现或消失
- 狭小空间里的触碰、视线、移动是否可能

输出矛盾列表和最小修复建议。
```
