# 物理连续性：防止人物瞬移和日常细节漂移

AI 长文写作常见的问题不是剧情忘了，而是物理状态忘了：

- 前排人物忽然出现在后排
- 后排人物突然能轻松碰到前排物品
- 人物刚才穿短裙，后面变成长裙
- 平底鞋无过渡变成高跟鞋
- 手机、包、信封、伤口突然消失
- 同一个位置/椅子/床/板凳/过道被塞进不合理数量的人
- 座椅突然变成床，桌子突然能站人，但中间没有转换动作

解决办法是把“空间”“物品”“承载资源”从普通剧情摘要里拆出来，单独维护物理状态账本。

## 推荐使用

生成前约束：

```powershell
python -m humanwriting.cli build `
  --style fiction `
  --strict-continuity `
  --review `
  --context examples/vehicle-scene-ledger.md `
  --task "续写车内争执。任何座位变化都必须写出动作过渡，保持服装和道具状态一致。"
```

成稿后审查：

```powershell
python -m humanwriting.cli audit `
  --draft examples/problem-car-scene-draft.zh-CN.md `
  --context examples/vehicle-scene-ledger.md `
  --profile physical
```

`--strict-continuity` 会自动加入：

- `spatial-blocking`
- `occupancy-capacity`
- `appearance-prop-continuity`

`physical-continuity-audit` 和证据表更严格的 `forensic-physical-audit` 只在 `audit --profile physical` 中加载，避免挤占正文生成的上下文。

## 必须记录什么

| 项目 | 示例 |
| --- | --- |
| 物理资源 | 座位、长椅、床、桌面、板凳、摩托车鞍座、飞机座位、过道、地面 |
| 当前形态 | 单人座、长椅、床、站立面、储物区、放倒、折叠、阻塞 |
| 容量规则 | 一人、多人长椅、可临时站立、仅能放物、未知 |
| 占用者/物品 | 谁坐着、躺着、站着、靠着，什么物件占着空间 |
| 可触达范围 | 能碰到谁/什么，是否隔着障碍 |
| 转换过渡 | 放倒座椅、清空桌面、停车换座、挪开包、爬上板凳 |

## 写作规则

- 没有移动过渡，就保持上一位置。
- 没有换装过渡，就保持上一服装。
- 没有放下/交出/丢失动作，道具仍在原处。
- 没有形态转换，就不能把单人座当床、把桌子当站台、把小板凳当多人平台。
- 容量未知时，先标记为未知，不要自动假定能容纳多人。
- 人物能不能看见、碰到、挡住别人，要由位置、形态、障碍和姿态决定。

## 审稿口令

可以在 Chatbox 或其他模型里输入：

```text
只做物理连续性审查，不要续写。

检查：
- 每个人当前在哪个位置
- 谁在前，谁在后，谁在左，谁在右
- 是否有人无过渡换座位
- 服装、鞋子、发型、伤口是否漂移
- 道具是否凭空出现或消失
- 狭小空间里的触碰、视线、移动是否可能
- 物理资源是否超容量
- 座位/床/板凳/桌面/过道是否发生形态转换
- 多人共享同一资源是否有姿态或拥挤描写

输出矛盾列表和最小修复建议。
```

## 为什么要用 `audit`

`build --strict-continuity` 适合生成前约束，但它不能保证模型会主动回头逐句查错。`audit --profile physical` 会生成“法医式审稿指令包”，强制模型先抽证据表，再判断矛盾。

它会重点抓：

- 前排/后排位置漂移
- 物理资源容量冲突
- 隔音玻璃、隔板、座椅、安全带造成的触达限制
- 明明没有停车/开门/换座，却发生近身接触
- 平底鞋变高跟鞋、短裙变长裙、外套凭空消失
- 手机、包、信封、钥匙等道具位置变化
